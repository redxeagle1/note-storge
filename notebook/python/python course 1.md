## الوحدة 1

### موضوع 1:-

إن أبسط تعريف للبرمجة هو أنها الطريقة التي يتواصل بها الأشخاص مع أجهزة الكمبيوتر ويطلبون منهم أداء المهام. عند الانتهاء من هذا الفيديو، ستتمكن من تحديد أساسيات البرمجة والبرمجة، ووصف كيف يمكن للبرمجة تقسيم المشكلات إلى خطوات أصغر، باستخدام التعليمات البرمجية لإيجاد الحلول ومناقشة كيفية تأثير البرمجة على الحياة اليومية، مثل التطبيقات والألعاب ومواقع الويب والسيارات ذاتية القيادة. يمكن تعريف البرمجة على أنها فن كتابة التعليمات للكمبيوتر لتنفيذها، وإخباره بما يجب القيام به، وكيف ومتى يفعل ذلك. تتم كتابة هذه التعليمات بلغة برمجة، مثل Python. أساسيات البرمجة هي العناصر الأساسية أو المفاهيم التي تتكون منها البرمجة، بغض النظر عن لغة البرمجة المحددة. تتضمن هذه الأساسيات المتغيرات وأنواع البيانات والمشغلين وبيانات تدفق التحكم والوظائف.

- **المتغيرات (Variables):** تشبه الحاويات التي تخزن البيانات في ذاكرة الكمبيوتر. تسمح لك بإعطاء أسماء للقيم، مما يسهل الرجوع إليها ومعالجتها في جميع أنحاء البرنامج، مما يجعل التعليمات البرمجية أكثر تنظيمًا وقابلية للقراءة.
- **أنواع البيانات (Data Types):** تحدد الأنواع المختلفة من البيانات التي يمكن للمتغير الاحتفاظ بها. تتضمن أنواع البيانات الشائعة في Python:
    - الأعداد الصحيحة (Integers): أرقام صحيحة مثل 0 أو 10 أو -5.
    - أرقام النقاط العائمة (Floating-point numbers): أرقام ذات نقاط عشرية مثل 3.14 أو -2.5.
    - السلاسل (Strings): نص محاط بعلامات اقتباس مثل "مرحبًا".
    - القيم المنطقية (Booleans): تمثيلات لقيم صحيحة (True) أو خاطئة (False).
- **عوامل التشغيل (Operators):** رموز تقوم بعمليات على البيانات. تتضمن العوامل الشائعة في Python:
    - معاملات حسابية (Arithmetic operators): للجمع والطرح والضرب والقسمة.
    - معاملات المقارنة (Comparison operators): مثل أكبر من، وأقل من، وتساوي، لمقارنة القيم.
    - عوامل التشغيل المنطقية (Logical operators): مثل AND و OR و NOT، لدمج الشروط.
- **عبارات التحكم في التدفق (Control Flow Statements):** تحدد الترتيب الذي يتم به تنفيذ التعليمات البرمجية. تشمل:
    - عبارات شرطية (Conditional statements): مثل IF أو ELIF أو ELSE، لتنفيذ مجموعات مختلفة من التعليمات البرمجية بناءً على شروط معينة.
    - الحلقات (Loops): لتكرار مجموعة من التعليمات البرمجية عدة مرات.
- **الوظائف (Functions):** كتل من التعليمات البرمجية قابلة لإعادة الاستخدام تؤدي مهام محددة. تجعل التعليمات البرمجية أكثر تنظيمًا وكفاءة وأسهل في الصيانة وتقلل التكرار.

تساعدك أساسيات البرمجة على تقسيم المشكلات إلى خطوات أصغر وأكثر قابلية للإدارة وكتابة تعليمات برمجية فعالة ومحسّنة.

مثال لاستخدام التعليمات البرمجية لحل المشكلات (تحليل مشتريات العملاء):

1. احصل على بيانات شراء العميل.
2. قم بتنظيف البيانات.
3. قم بتحويل البيانات للتحليل.
4. حدد الاتجاهات أو الأنماط أو الرؤى في البيانات.
5. اعرض النتائج. يتم استخدام كود Python في كل خطوة، مع الاستفادة من المتغيرات والحلقات والعبارات الشرطية والوظائف والمكتبات مثل pandas و matplotlib.

البرمجة تحيط بنا في كل مكان، من تطبيقات الهاتف المحمول ووسائل التواصل الاجتماعي والألعاب وأدوات الإنتاجية إلى مواقع الويب والمتاجر عبر الإنترنت ومحركات البحث. كما تتحكم في إشارات المرور والأدوات الجراحية والمركبات ذاتية القيادة.

### موضوع 2:- (محادثة ثنائية الأطراف حول صعوبة التعامل مع كود موقع ويب قديم واقتراح استخدام Python)

يناقش شخصان صعوبة صيانة كود موقع ويب قديم. يقترح أحدهما استخدام Python نظرًا لكفاءتها وقابليتها للصيانة. Python متعددة الاستخدامات وتستخدم في تطوير الويب (مثل إطار Django الذي تستخدمه بعض الشركات للتعامل مع ملايين تفاعلات المستخدمين) وفي علوم البيانات والتعلم الآلي والرعاية الصحية (لتحليل الصور الطبية). تتميز Python بصيغة سهلة القراءة والكتابة، مما يسرع عملية التغيير. يمكن استخدام Python لأتمتة المهام المتكررة. يقررون تجربة Python لتحديث الموقع.

### موضوع 3:- (مقدمة إلى لغة Python وتاريخها وفلسفتها وخصائصها الرئيسية)

أصبحت Python أداة برمجة شائعة ومفيدة بسبب تركيبتها وقابليتها للقراءة والمكتبات. ستتعرف في هذا القسم على Python وتاريخها وفلسفتها وخصائصها الرئيسية، وقدرتها على دعم البرمجة الموجهة للكائنات (Object-Oriented Programming - OOP).

تم إنشاء Python في أواخر الثمانينيات بواسطة Guido van Rossum. ركزت فلسفته على سهولة القراءة والبساطة، مما يجعلها لغة سهلة التعلم، حتى للمبتدئين، مع توفير الوظائف اللازمة للمشاريع المعقدة. تتميز Python بثلاث خصائص مميزة:

- **لغة مفسرة (Interpreted Language):** يتم تنفيذ كود Python سطرًا بسطر بواسطة مترجم، مما يسمح برؤية النتائج فورًا دون خطوة تجميع منفصلة.
- **لغة عالية المستوى (High-Level Language):** تبسط Python العديد من تعقيدات اللغات ذات المستوى الأدنى، مما يسمح بالتركيز على منطق البرنامج.
- **البرمجة الشيئية (Object-Oriented Programming - OOP):** تدعم Python البرمجة الموجهة للكائنات، وهو نموذج ينظم التعليمات البرمجية حول هياكل البيانات التي تحتوي على بيانات ووظائف (كائنات).

تعدد استخدامات Python هو أحد نقاط قوتها. يمكن استخدامها في:

- **تطوير الويب (Web Development):** أطر عمل مثل Django و Flask. Django يوفر ميزات مدمجة، بينما Flask يوفر مرونة أكبر.
- **علوم البيانات والتعلم الآلي (Data Science and Machine Learning):** مكتبات مثل NumPy و Pandas و Scikit-Learn.
- **الأتمتة والبرمجة النصية (Automation and Scripting):** لأتمتة عمليات سير العمل وإنشاء نصوص مخصصة.
- تطبيقات أخرى تشمل تطوير الألعاب والحوسبة العلمية والتعليم.

تمتلك Python مجتمعًا افتراضيًا مزدهرًا من المستخدمين الذين يشاركون معارفهم وخبراتهم.

### موضوع 4:- (جمال بناء جملة Python وأنواع البيانات ونقاط قوة Python للمبتدئين)

تعطي Python الأولوية للصيغة النظيفة وقابلية القراءة، وغالبًا ما يشبه رمزها اللغة الإنجليزية البسيطة. على الرغم من طبيعتها الصديقة للمبتدئين، فإن تعدد استخداماتها ومكتباتها الواسعة تسمح بمعالجة المشاريع المعقدة.

#### جمال بناء جملة Python

تعتمد Python على **المسافة البادئة (Indentation)** (استخدام المسافات أو علامات التبويب) لتحديد بنية التعليمات البرمجية، بدلاً من الفواصل المنقوطة أو الأقواس المتعرجة. هذا يجعل الكود يبدو أشبه باللغة الإنجليزية العادية ويفرض أسلوبًا متسقًا ومنظمًا.

مثال لرسالة ترحيب بسيطة:

Python

```
print("Hello, world!")
```

مثال يوضح أهمية المسافة البادئة في العبارات الشرطية:

Python

```
if grade == "A":
    print("Excellent work!")
else:
    print("Keep studying and you'll improve!")
```

المسافة البادئة هنا تحدد الكتل البرمجية التابعة لعبارات `if` و `else`. الكود بدون مسافة بادئة صحيحة قد يسبب أخطاء أو يكون صعب الفهم.

#### أنواع البيانات: اللبنات الأساسية للكود الخاص بك

أنواع البيانات تشبه الأدوات المختلفة، كل منها مصمم لغرض معين.

- **الأرقام (Numbers):** لتخزين القيم الرقمية مثل العمر (25) أو السعر (19.99). تستخدم **الأعداد الصحيحة (Integers)** للأرقام الصحيحة وأرقام **الفاصلة العائمة (Floating-point numbers)** للأعداد العشرية.
- **السلاسل (Strings):** للعمل مع نص، مثل اسم شخص ("أليس") أو عنوان كتاب.
- **القيم المنطقية (Booleans):** لتمثيل الحالات المنطقية مثل صح (True) أو خطأ (False).
- **القوائم (Lists):** لتخزين مجموعات مرتبة من العناصر، مثل قائمة البقالة.
- **القواميس (Dictionaries):** لربط القيم بمفاتيح، مثل تخزين معلومات المستخدم (الاسم، العمر، البريد الإلكتروني).

#### نقاط قوة Python للمبتدئين

Python مثالية للمبتدئين بسبب سهولة قراءتها التي تشبه اللغة الإنجليزية العادية، مما يقلل عامل التخويف ويعزز فهم مفاهيم البرمجة. مجتمع Python الداعم يوفر الكثير من البرامج التعليمية والموارد. تعدد استخدامات Python يسمح للمبتدئين باستكشاف مجالات برمجة متنوعة مثل تطوير الويب وتحليل البيانات والأتمتة والتعلم الآلي. نظامها البيئي الغني بالمكتبات والأطر يوفر أدوات معدة مسبقًا لإنجاز المهام المعقدة بأقل جهد. ظهور أدوات الذكاء الاصطناعي التوليدية (Generative AI tools) المدمجة في بيئات تطوير Python تُحدث ثورة في كيفية عمل المبرمجين من خلال:

- **إنشاء التعليمات البرمجية (Code generation):** أدوات إكمال التعليمات البرمجية التي تعمل بالذكاء الاصطناعي مثل GitHub Copilot تقترح مقتطفات التعليمات البرمجية.
- **المساعدة في تصحيح الأخطاء (Debugging assistance):** أدوات تصحيح الأخطاء التي تعمل بالذكاء الاصطناعي تحلل التعليمات البرمجية بحثًا عن الأخطاء وتقترح الإصلاحات.
- **المساعدة على التعلم (Learning aid):** يمكن للذكاء الاصطناعي التوليدي أن يعمل كمدرس افتراضي.
- **الإلهام الإبداعي (Creative inspiration):** يمكن للذكاء الاصطناعي التوليدي إطلاق أفكار جديدة واقتراح طرق بديلة لحل المشكلات.

Python تحقق توازنًا بين القوة والبساطة، مما يجعلها الرفيق المثالي للمبتدئين.

### موضوع 5:- (استخدامات Python في تطبيقات العالم الحقيقي)

يُطلق على Python أحيانًا اسم "سكين الجيش السويسري" للغات البرمجة بسبب قدرتها على التكيف في تطبيقات الويب، معالجة البيانات، والذكاء الاصطناعي.

استخدامات Python تشمل:

- **تطوير الويب (Web Development):** أطر عمل مثل Django (يركز على التطوير السريع والتصميم النظيف) و Flask (إطار صغير يوفر أساسيات مع مزيد من التحكم). مواقع تستخدم Django تشمل Instagram، Disqus، و Spotify.
- **علوم البيانات (Data Science):** تتضمن مهام مثل جمع البيانات، تنظيفها، تحليلها، وتصورها. مكتبات Python المستخدمة:
    - Pandas: لتحليل البيانات ومعالجة مجموعات البيانات الكبيرة.
    - NumPy: للحوسبة الرقمية والعمل مع المصفوفات.
    - Matplotlib: لتصور البيانات باستخدام المخططات والرسوم البيانية.
- **الأتمتة والبرمجة النصية (Automation and Scripting):** لأتمتة المهام المتكررة مثل إعادة تسمية الملفات، تنظيم الملفات، أو استخراج البيانات. تتضمن تجريف الويب (Web scraping) باستخدام مكتبات مثل BeautifulSoup و Scrapy، وأتمتة البريد الإلكتروني.
- **التعلم الآلي (Machine Learning):** مجال يمنح أجهزة الكمبيوتر القدرة على التعلم دون برمجة صريحة. مكتبات Python المستخدمة:
    - Scikit-learn: لمهام التعلم الآلي مثل التصنيف والانحدار.
    - Microsoft Cognitive Toolkit و Keras: مكتبات للتعلم العميق لبناء وتدريب الشبكات العصبية لتطبيقات مثل التعرف على الصور ومعالجة اللغات الطبيعية.
- **مجالات أخرى:**
    - تطوير الألعاب (Game development): مكتبات مثل Pygame للألعاب ثنائية الأبعاد.
    - الحوسبة العلمية (Scientific computing): قدرات Python العددية ومكتبات مثل SciPy للبحث العلمي والمحاكاة.
    - التعليم (Education): بساطة Python وقابليتها للقراءة تجعلها لغة أولى جيدة لتدريس مفاهيم البرمجة.

أسباب اختيار Python:

1. سهولة القراءة والتعلم.
2. مجموعة واسعة من المكتبات والأطر.
3. مجتمع نشط وداعم.
4. التوافق عبر الأنظمة الأساسية (Cross-platform compatibility).

### موضوع 6:- (مجموعة أدوات Python للمبتدئين)

(محادثة بين شخصين حول الأدوات اللازمة لتعلم Python)

قد تبدو الخيارات الكثيرة مربكة للمبتدئين. "مجموعة أدوات Python" هي دليل للبدء خطوة بخطوة.

تتضمن مجموعة الأدوات:

- **إعداد بيئة Python:** التأكد من تثبيت الإصدار الصحيح.
- **اختيار بيئة التطوير المتكاملة (IDE - Integrated Development Environment):** وهي مساحة عمل الترميز الشخصية، تشبه معالجات النصوص ولكنها مصممة لكتابة التعليمات البرمجية.
- **أدوات تنظيم المشاريع:** مثل البيئات الافتراضية (Virtual Environments)، وهي بمثابة حاويات منفصلة لكل مشروع للحفاظ على كل شيء منظمًا ومنع التعارضات.
- **مكتبات Python (Python Libraries):** مجموعات من التعليمات البرمجية المكتوبة مسبقًا يمكنك استخدامها في مشاريعك لتوفير الوقت والجهد. توجد مكتبات لإنشاء مواقع الويب، تحليل البيانات، إنشاء الألعاب، العمل مع الذكاء الاصطناعي، وأتمتة المهام.
- **Pip:** أداة (package installer) لتثبيت وإدارة هذه المكتبات بسهولة.

تهدف هذه الأدوات إلى مساعدة المتعلمين على أن يصبحوا محترفين في Python، قادرين على كتابة التعليمات البرمجية، إدارة المشاريع، والمساهمة في مجتمع المصادر المفتوحة.

### موضوع 7:- (تثبيت Python باستخدام Anaconda)

دليل لتثبيت Python على جهاز الكمبيوتر باستخدام توزيع Anaconda. Anaconda هو خيار شائع لعلوم البيانات والحوسبة العلمية لأنه يأتي محملاً مسبقًا بالعديد من الحزم المفيدة ويبسط إدارة الحزم. سيتم أيضًا تغطية كيفية تثبيت حزم إضافية باستخدام "pip".

#### لماذا Anaconda؟

مزايا Anaconda:

- **إدارة الحزم الشاملة:** تأتي مع مجموعة كبيرة من الحزم المثبتة مسبقًا.
- **إدارة بيئة Conda:** تسهل أداة conda إنشاء وإدارة بيئات معزولة للمشاريع.
- **التوافق عبر الأنظمة الأساسية:** تعمل على Windows و macOS و Linux.
- **واجهة سهلة الاستخدام:** يوفر Anaconda Navigator واجهة رسومية لإدارة البيئات وتشغيل التطبيقات.

#### مكونات Python الأساسية التي يتم تثبيتها:

- **مترجم Python (Python Interpreter):** هو جوهر Python. يقرأ الكود سطرًا بسطر، يترجمه إلى تعليمات تفهمها الآلة، وينفذها.
    
    - الإيجابيات: قابلية النقل (Portability) عبر المنصات، والوضع التفاعلي (Interactive mode) للتجريب.
    - السلبيات: كلغة مفسرة، قد تكون أبطأ من اللغات المترجمة لبعض المهام.
- **المكتبة القياسية (Standard Library):** مجموعة كبيرة من الوحدات (كود مكتوب مسبقًا) تأتي مرفقة مع Python وتوفر وظائف لمجموعة واسعة من المهام.
    
    - الإيجابيات: الراحة والموثوقية.
    - السلبيات: نطاق محدود؛ قد تحتاج إلى تثبيت مكتبات إضافية تابعة لجهات خارجية للاحتياجات المتخصصة.
- **Pip (Pip Installs Packages):** أداة سطر أوامر لتثبيت وإدارة الحزم الخارجية من فهرس حزم Python (PyPI). يحتوي PyPI على آلاف المكتبات والأطر.
    
    - الإيجابيات: نظام بيئي واسع وسهولة الاستخدام.
    - السلبيات: إدارة التبعية يمكن أن تؤدي أحيانًا إلى تعارضات.

#### خطوات تثبيت Anaconda:

1. **تنزيل Anaconda:** انتقل إلى [صفحة تنزيلات Anaconda الرسمية](https://www.anaconda.com/download). أدخل بريدك الإلكتروني أو استخدم رابط تخطي التسجيل. حدد زر التنزيل لنظام التشغيل الخاص بك.
2. **تشغيل برنامج التثبيت:**
    - **Windows:** انقر نقرًا مزدوجًا فوق الملف `.exe` واتبع التعليمات.
    - **Mac:** انقر نقرًا مزدوجًا فوق الملف `.pkg`، اسمح بالإذن، واتبع التعليمات.
    - **Linux:** قم بتحديث قائمة الحزم (`sudo apt-get update`)، قم بتثبيت التبعيات اللازمة (راجع [موقع مستندات Anaconda](https://www.google.com/search?q=https://docs.anaconda.com/anaconda/install/linux/%23prerequisites) للحصول على أحدث التبعيات)، قم بتعيين أذونات التنفيذ للملف `.sh` (`chmod +x Anacondafilename.sh`)، قم بتنفيذ الملف (`./Anacondafilename.sh`)، واقبل اتفاقية الترخيص واتبع التعليمات. قد يُسأل عما إذا كنت تريد تهيئة Anaconda3 عن طريق تشغيل `conda init`.
3. **التحقق من التثبيت:**
    - **Windows:** افتح "Anaconda Prompt" من قائمة ابدأ، اكتب `python --version` و `conda --version`. يمكنك أيضًا تشغيل Anaconda Navigator.
    - **Mac:** افتح Terminal، اكتب `conda --version` و `python --version`. تحقق من مسار Anaconda باستخدام `echo $PATH`.
    - **Linux:** أغلق وأعد فتح الجهاز الطرفي أو قم بتشغيل `source ~/.bashrc`. اكتب `conda --version` و `python --version`.

**نصائح عامة وأفضل الممارسات بعد التثبيت:**

- انتبه لرسائل الخطأ.
- قم بتنزيل أحدث إصدار مستقر من الموقع الرسمي.
- تحقق من سلامة المثبت (SHA256 hash).
- قم بالتثبيت في دليل المستخدم إن أمكن.
- تأكد من وجود اتصال ثابت بالإنترنت.
- قم بتشغيل `conda update conda` و `conda update anaconda` بعد التثبيت.

**لماذا تحتاج إلى Jupyter Notebook؟**

- **التوثيق والمشاركة (Documentation and Sharing):** يسمح بمزج التعليمات البرمجية والمخرجات والنص التوضيحي في مستند واحد. يمكن استخدام خلايا Markdown لإنشاء نص منسق ودمج الصور ومقاطع الفيديو.
- **التجريب والنماذج الأولية (Experimentation and Prototyping):** الطبيعة التفاعلية تعزز الاستكشاف والتجريب، مما يجعلها أداة جيدة للنماذج الأولية وتطوير الخوارزميات.

**سيناريوهات العالم الحقيقي لاستخدام Jupyter Notebook:**

- **علم البيانات (Data Science):** استيراد بيانات العملاء وتنظيفها، إجراء تحليل استكشافي، بناء نماذج تنبؤية، وتصور النتائج.

### موضوع 12:- (العمليات الحسابية والمتغيرات في Python)

أسباب استخدام Python للمهام الحسابية: السرعة والكفاءة، الدقة، الأتمتة، والمرونة.

**العمليات الحسابية الأساسية:**

- **الإضافة (+):** لدمج قيم أو حساب الإجماليات.

### موضوع 17:- (العبارات والتعبيرات والأتمتة في Python)

الفهم القوي للمكونات الأساسية لبرنامج Python (العبارات، التعبيرات، والأتمتة) هو المفتاح لإنشاء كود فعال ومحسّن.

#### العبارات (Statements): الأوامر التي تنظم الكود

العبارات هي أسطر فردية من التعليمات البرمجية تملي الإجراءات التي يجب أن يقوم بها الكمبيوتر.

مثال:

Python

```
age = 25
if age >= 18:
    print("You are eligible to vote.")
else:
    print("You are not yet eligible to vote.")
```

أنواع العبارات في المثال:

- **بيان التعيين (Assignment Statement):** `age = 25` يعين القيمة للمتغير.
- **عبارات شرطية (Conditional Statements):** `if age >= 18:` و `else:` تنشئ مسارات متفرعة.
- **عبارات الطباعة (Print Statements):** `print(...)` تعرض الرسائل للمستخدم.

**أنواع إضافية من العبارات:**

- **عبارات التكرار (Iteration Statements):** مثل `for` و `while`، لأتمتة المهام المتكررة.
- **تعريفات الوظائف (Function Definitions):** إنشاء كتل قابلة لإعادة الاستخدام من التعليمات البرمجية.
- **تعريفات الفئة (Class Definitions):** أساس البرمجة الموجهة للكائنات، تعمل كمخططات لإنشاء الكائنات.

#### التعبيرات (Expressions): الحسابات التي تدعم منطق الكود

التعبيرات هي الحسابات التي تحدث داخل العبارات وتنتج قيمًا.

العمليات الحسابية الأساسية: الجمع (+)، الطرح (-)، الضرب (*)، القسمة (/).

عمليات حسابية أقل تكرارًا:

- **القسمة الأرضية (Floor Division `//`):** تُجري القسمة وتتخلص من أي متبقٍ (مثال: `7 // 4` هي 1).
- **Modulo (`%`):** تُرجع النسبة المتبقية بعد القسمة الصحيحة (مثال: `7 % 4` هي 3).
- **الأسس (`**`):** ترفع رقمًا إلى قوة رقم آخر (مثال: `2 ** 4` هي 24).

مثال:

Python

```
discount = 0.2  # 20% discount
price = 50
final_price = price * (1 - discount)
print(final_price)
```

في هذا الكود:

- `0.2` و `50` هي تعبيرات بسيطة.
- `1 - discount` هو تعبير أكثر تعقيدًا يحسب نسبة الخصم.
- `price * (1 - discount)` هو التعبير النهائي الذي يحدد `final_price`.

قدرة Python على التشغيل الآلي هي واحدة من أكثر سماتها إلحاحًا.

مثال: تحليل آلاف نقاط البيانات التجريبية. يمكن لبرنامج نصي Python أتمتة العملية بأكملها:

    - **الاستخراج (Extraction):** قراءة البيانات ذات الصلة من كل ملف باستخدام مكتبات مثل Pandas. مثال بسيط لملف CSV:

    - **التصور (Visualization):** إنشاء رسوم بيانية ومخططات (مكتبات مثل Matplotlib، Apache Superset).
    - **التقرير (Reporting):** تلخيص النتائج (أدوات مثل Jupyter Notebooks).

بيان `if` هو بيان القرار الأساسي والأكثر استخدامًا[cite: 780]. يقوم بتقييم شرط معين، وإذا كان الشرط صحيحًا، يتم تنفيذ مجموعة التعليمات البرمجية المرتبطة به[cite: 781, 782]. إذا كان الشرط خاطئًا، يتم تخطي تلك المجموعة من التعليمات البرمجية[cite: 783].

```python
order_total = 60
```

```python
order_total = 30
```

```python
loyalty_points = 150
```

#### مقدمة إلى الحلقات والعبارات الشرطية

تعتبر الحلقات والعبارات الشرطية لبنات أساسية في البرمجة، بما في ذلك Python، وتسمح بتنفيذ التعليمات البرمجية بكفاءة وديناميكية[cite: 833, 834].

###### الحلقات: قوة التكرار

توفر الحلقات طريقة لتنفيذ التعليمات بشكل متكرر، مما يجنب التكرار ويعزز الكفاءة[cite: 836, 837]. تقدم Python نوعين أساسيين من الحلقات: `for` و `while`[cite: 839].

###### حلقة `for`: التكرار عبر التسلسلات

تتكرر حلقة `for` عبر تسلسل (مثل قائمة أو سلسلة) وتنفذ مجموعة من التعليمات البرمجية لكل عنصر في التسلسل[cite: 840]. هذا مفيد عندما يكون عدد التكرارات معروفًا مسبقًا[cite: 841].

مثال لطباعة الأرقام من 1 إلى 10:

```python
for i in range(1, 11):
  print(i)
```

في هذا المثال، تنشئ `range(1, 11)` تسلسلاً من الأرقام من 1 إلى 10[cite: 843]. يأخذ متغير الحلقة `i` كل قيمة في التسلسل، ويتم تنفيذ مجموعة التعليمات البرمجية داخل الحلقة لكل قيمة[cite: 843]. لا تقتصر حلقات `for` على الأرقام فقط؛ يمكن استخدامها لمعالجة قوائم الأسماء، أو حساب إجمالي العناصر، أو تحليل سلاسل الأحرف[cite: 844, 845, 846].

###### حلقة `while`: التنفيذ بناءً على الظروف

تنفذ حلقة `while` مجموعة من التعليمات البرمجية طالما ظل الشرط المحدد صحيحًا[cite: 847]. هذا مفيد عندما يكون عدد التكرارات غير معروف مسبقًا أو عندما تحتاج الحلقة إلى الاستمرار حتى يتم استيفاء شرط معين[cite: 848].

مثال يطالب المستخدم بالإدخال حتى يتم إدخال رقم أكبر من 0:

```python
valid_input = False
while not valid_input:
  user_input = int(input("Please enter a number greater than 0: "))
  if user_input > 0:
    valid_input = True
  else:
    print("Invalid input. Please try again.")
```

تستمر هذه الحلقة في التنفيذ حتى يصبح متغير `valid_input` صحيحًا، مما يضمن حصول البرنامج على الإدخال المطلوب[cite: 852]. هذا النوع من الحلقات مفيد في الألعاب التفاعلية أو المحاكاة أو معالجة البيانات في الوقت الفعلي[cite: 853].

###### بيان `if-else`: التنفيذ المتفرّع

### موضوع 3:-

العبارات الشرطية هي حراس المنطق في Python، تقوم بتقييم شرط معين، وبناءً على ما إذا كان هذا الشرط صحيحًا أم خاطئًا، تقوم بتنفيذ مجموعات مختلفة من التعليمات البرمجية[cite: 912, 913, 914]. إنها تمكّن البرامج من التكيف والاستجابة واتخاذ خيارات ذكية بناءً على ظروف مختلفة[cite: 915, 916].

هناك ثلاث عبارات شرطية أساسية في Python:

  * **`if`**: تتحقق من شرط في الكود. إذا كان الشرط صحيحًا، يتم تنفيذ مجموعة التعليمات البرمجية داخل عبارة `if`. إذا كان خاطئًا، يتم تخطي مجموعة التعليمات البرمجية هذه[cite: 918, 919, 920]. مثال: التحقق مما إذا كانت مبيعات الموظف تتجاوز حدًا معينًا لتشغيل وظيفة حساب المكافأة[cite: 921, 922].
  * **`else`**: تقترن بعبارة `if` وتوفر مجموعة تعليمات برمجية بديلة ليتم تنفيذها إذا كان شرط `if` خاطئًا[cite: 923, 924]. مثال: في نظام تسجيل الدخول، إذا كانت بيانات الاعتماد صالحة (`if`)، يتم منح الوصول؛ وإلا (`else`)، يتم عرض رسالة خطأ[cite: 925, 926, 927].
  * **`elif`**: اختصار لـ "else if"، وتُستخدم لمعالجة أكثر من خيار واحد[cite: 928]. تسمح بالتحقق من شروط متعددة بالتسلسل[cite: 929, 930]. مثال: حساب تكاليف الشحن في عربة التسوق عبر الإنترنت بناءً على إجمالي الطلب، حيث يمكن تطبيق أسعار شحن مختلفة لنطاقات أسعار مختلفة[cite: 931, 932].

تعد العبارات الشرطية العمود الفقري للبرامج الديناميكية القابلة للتكيف للأسباب التالية[cite: 933]:

  * **التعامل مع سيناريوهات مختلفة**: تسمح للكود بالاستجابة بشكل مناسب لإدخال المستخدم وتقلبات البيانات والأحداث الخارجية[cite: 934, 935].
  * **إنشاء أشجار قرارات معقدة**: تمكّن من تقييم شروط متعددة وتنفيذ إجراءات محددة بناءً على النتائج، وهو أمر مهم للتحقق من صحة البيانات ومعالجة الأخطاء وتنفيذ قواعد اللعبة[cite: 936, 937].
  * **إنشاء تجارب مخصصة**: يمكنها تخصيص مخرجات البرنامج أو وظائفه للمستخدمين الفرديين بناءً على تفضيلاتهم أو تحليل البيانات أو عوامل أخرى ذات صلة[cite: 938, 939].

أمثلة على تطبيقات العالم الحقيقي:

  * **منطق اللعبة**: تحديد كيفية استجابة اللعبة لإجراءات اللاعب، مثل تشغيل رسالة "انتهت اللعبة" عندما تنخفض نقاط صحة اللاعب[cite: 941, 942].
  * **التحقق من صحة البيانات**: التأكد من أن إدخال المستخدم يفي بمعايير محددة، مثل التحقق من صحة عنوان البريد الإلكتروني أو متطلبات تعقيد كلمة المرور[cite: 943, 944].
  * **معالجة الأخطاء**: تمكين البرامج من توقع الأخطاء المحتملة أو المواقف غير المتوقعة ومعالجتها، مثل عرض رسالة خطأ مناسبة إذا كان الملف الذي يحاول البرنامج الوصول إليه غير موجود[cite: 945, 946].

##### العبارات والحلقات الشرطية: أساس تدفق البرنامج

يحدد تدفق التحكم خطوط التعليمات البرمجية التي يتم تنفيذها، وبأي ترتيب، وتحت أي ظروف[cite: 1035]. تعتبر **العبارات الشرطية** **والحلقات** من أهم الأدوات الأساسية في صندوق أدوات التحكم في التدفق[cite: 1039]. تسمح العبارات الشرطية للكود باتخاذ القرارات واختيار مسارات مختلفة بناءً على الشروط[cite: 1041, 1042]. تسمح الحلقات للكود بتكرار مجموعة من التعليمات عدة مرات[cite: 1043].

###### لماذا يعد التحكم في التدفق مهمًا

بدون التحكم في التدفق، لن تتمكن برامج Python من القيام بأكثر من تشغيل سلسلة بسيطة من التعليمات[cite: 1044]. يمكن للتحكم في التدفق التحقق من صحة الإدخال، والتحقق من التكرارات، وتقديم ملاحظات للمستخدم[cite: 1049].

###### عبارات شرطية: اتخاذ الخيارات

تعمل العبارات الشرطية كمنطق "if-then-else" للبرمجة، مما يمكّن الكود من تقييم الموقف وتحديد مسار العمل[cite: 1050, 1051]. حجر الزاوية هو عبارة `if`[cite: 1054].

مثال كود لدرجة الحرارة:

```python
temperature = 22  # Example temperature
if temperature < 20:
    print("It might be cold! You might need a jacket.")
else:
    print("It's warm! Enjoy the sunshine.")
```

يمكن تحسين عملية صنع القرار بدمج `elif` (اختصار لـ "else if") لتقييم شروط متعددة بالتتابع[cite: 1056].

مثال كود لدرجة الحرارة مع `elif`:

```python
temperature = 15  # Let's assume the temperature is 15 degrees Celsius
if temperature < 10:
    print("Wear a jacket! It's cold out there.")
elif 10 <= temperature < 20:
    print("A light sweater should be fine. It's a bit chilly.")
elif 20 <= temperature < 30:
    print("Enjoy the pleasant weather! No need for extra layers.")
else:
    print("It's hot! Stay hydrated and wear sunscreen.")
```

###### Loops: التكرار التلقائي

تسمح الحلقات بتنفيذ مجموعة من التعليمات البرمجية عدة مرات، إما لعدد محدد من التكرارات أو حتى يتم استيفاء شرط معين[cite: 1060]. تقدم Python نوعين أساسيين: `for` و `while`[cite: 1061].

حلقات `for` مثالية عندما تعرف عدد التكرار المطلوب مسبقًا[cite: 1061].

```python
# Example: Print numbers from 1 to 5
```

```python
# Example: Keep asking for input until the user enters "quit"
```

### موضوع 6:-

تتبع تنفيذ التعليمات البرمجية في Python أمر ضروري لفهم كيفية عمل البرنامج وتصحيح الأخطاء[cite: 1093]. الفوائد الرئيسية للتتبع تشمل المساعدة في تصحيح الأخطاء، وفهم منطق الكود، وتحسين الأداء[cite: 1095].

###### مكونات التتبع:

  * **تصحيح الأخطاء (Debugging)**: عند حدوث خطأ، يتيح لك التتبع تحديد السطر الدقيق أو الوظيفة التي يحدث فيها الخطأ[cite: 1096].
  * **فهم الكود (Code Comprehension)**: يساعد التتبع في فك رموز منطق البرامج المعقدة، خاصة عند التعامل مع كود كتبه شخص آخر[cite: 1098, 1099].
  * **التحسين (Optimization)**: من خلال مراقبة كيفية معالجة المترجم للكود وإدارة الذاكرة، يمكن كتابة كود أكثر كفاءة[cite: 1107, 1108].

###### عناصر أساسية في عملية التنفيذ:

  * **نص Python (Python Script)**: تسلسل التعليمات المكتوبة[cite: 1104].
  * **المترجم (Interpreter)**: يقرأ الكود سطرًا بسطر وينفذ التعليمات، مترجمًا الكود عالي المستوى إلى تعليمات يمكن للآلة فهمها[cite: 1105, 1106, 1107].
  * **مكدس الاستدعاءات (Call Stack)**: بنية بيانات تتعقب استدعاءات الوظائف النشطة ونقاط عودتها، مما يساعد على فهم تدفق البرنامج وتفاعل الوظائف[cite: 1109, 1110, 1111].
  * **المتغيرات (Variables)**: ذاكرة البرنامج التي تخزن القيم[cite: 1112, 1113]. يتيح التتبع مراقبة كيفية تغير قيم المتغيرات[cite: 1114].

###### أداة التتبع الأساسية: عبارة `print()`

تُستخدم عبارات `print()` المدرجة في مواقع استراتيجية من أجل[cite: 1116]:

  * **فحص المتغيرات**: لمعرفة كيف تتغير قيمها بمرور الوقت[cite: 1117].
  * **تتبع تدفق التنفيذ**: للتحقق من ترتيب استدعاء الوظائف[cite: 1117].
  * **اكتشاف الأخطاء المخفية**: طباعة القيم يمكن أن تكشف عن نتائج غير متوقعة تشير إلى أخطاء[cite: 1118].

مثال لوظيفة بسيطة وتتبع تنفيذها:

```python
def greet(name):
    message = "Hello" + name
    return message

print(greet("Alice"))
```

التتبع:

1.  تحديد الدالة `greet` (لا يتم تنفيذها بعد)[cite: 1119].
2.  استدعاء الدالة `print`، وداخلها يتم استدعاء الدالة `greet` بالوسيط `Alice`[cite: 1120].
3.  داخل `greet`، يتم تعيين المتغير `message` إلى `"HelloAlice"`[cite: 1121].
4.  عبارة `return message` تعيد هذه القيمة إلى مكان استدعاء الدالة[cite: 1121].
5.  تتلقى الدالة `print` القيمة `"HelloAlice"` وتعرضها[cite: 1122].

```python
    def inner_function():

        print(f"z يساوي {z}")

    print(f"y تساوي {y}")

outer_function(10)
```

##### الأخطاء الشائعة في تنفيذ التعليمات البرمجية: كيفية تجنبها

حتى مع طبيعة Python سهلة الاستخدام، هناك أخطاء شائعة يمكن أن تحدث أثناء تنفيذ الكود[cite: 1132, 1133]. فهم هذه الأخطاء وكيفية تجنبها أمر ضروري لكتابة كود Python فعال وموثوق[cite: 1137, 1138].

###### فهم تنفيذ التعليمات البرمجية:

يقوم مترجم Python بمسح الكود سطرًا بسطر من الأعلى إلى الأسفل[cite: 1141]. يقوم بتقسيم كل سطر إلى مكوناته الأساسية (كلمات رئيسية، متغيرات، عوامل تشغيل)، ثم يترجم هذه المكونات إلى لغة يمكن للكمبيوتر فهمها، وأخيرًا ينفذ هذه التعليمات[cite: 1142, 1143, 1144, 1145, 1146].

###### أخطاء شائعة وحلولها:

1.  **أخطاء المسافة البادئة (Indentation Errors)**

      * **المشكلة**: تعتمد Python على المسافة البادئة لتحديد بنية الكود. المسافة البادئة غير الصحيحة يمكن أن تؤدي إلى سلوك غير متوقع أو أخطاء صريحة[cite: 1150, 1151, 1152].
        ```python
        # مثال على خطأ في المسافة البادئة (الكود الأصلي يحتوي على الخطأ)
        x = 6
        if x > 5:
            print("x is greater than 5")
        # else: # Incorrect indentation in original example
        # print("x is less than or equal to 5")
        else: # Corrected indentation
            print("x is less than or equal to 5")
        ```
      * **الحل**: انتبه دائمًا للمسافة البادئة. تأكد من أن العبارات داخل الحلقات والشروط والوظائف والفئات محاذاة بشكل صحيح[cite: 1154]. معظم محررات الكود توفر إشارات مرئية للمساعدة[cite: 1155].

2.  **نطاق المتغير (Variable Scope)**

      * **المشكلة**: المتغيرات لها نطاق يحدد أين يمكن الوصول إليها. محاولة الوصول إلى متغير خارج نطاقه تؤدي إلى `NameError`[cite: 1156, 1157].
        ```python
        # مثال على مشكلة نطاق المتغير (الكود الأصلي يحتوي على الخطأ المحتمل)
        flag = False
        if flag:
            x = 10
            print(x) # No problem here
        # print(x) # NameError if flag = False in original example
        if flag == False: # Added condition to avoid error for demo
             print("x is not defined because flag is False")
        elif 'x' in locals() or 'x' in globals():
             print(x)

        ```
        في هذا المثال، يتم تعريف `x` داخل عبارة `if`. إذا كانت `flag` خاطئة، فإن محاولة طباعة `x` خارج عبارة `if` ستؤدي إلى خطأ[cite: 1158, 1159].
      * **الحل**: ضع في اعتبارك مكان تعريف المتغيرات. إذا كنت بحاجة إلى متغير ليكون متاحًا في جميع أنحاء الكود، فحدده في النطاق العام، ولكن استخدم المتغيرات العامة بحكمة لتجنب جعل الكود صعب الصيانة[cite: 1160, 1161, 1162].

3.  **القيم غير القابلة للتغيير (Immutable Values)**

      * **المشكلة**: السلاسل في Python غير قابلة للتغيير، مما يعني أنه لا يمكن تغيير محتوياتها بعد إنشائها[cite: 1163]. محاولة تعديل حرف في سلسلة مباشرة تؤدي إلى `TypeError`[cite: 1164, 1165].
        ```python
        # value = "Hello"
        # value[0] = 'h' # This would cause a TypeError
        # print(value)
        ```
      * **الحل**: عند "تغيير" سلسلة، فأنت تقوم فعليًا بإنشاء سلسلة جديدة[cite: 1166].
        ```python
        value = "Hello"
        new_value = "h" + value[1:]
        print(new_value)  # Output: hello
        ```

4.  **إعادة تعيين المتغير (Variable Reassignment)**

      * **المشكلة**: يمكن أن يؤدي استخدام نفس اسم المتغير مرة أخرى عن طريق الخطأ إلى تغيير قيمته الأصلية، مما قد يؤدي إلى نتائج غير متوقعة[cite: 1169, 1170].
        ```python
        x = 5
        if True:
           x = 10  # You've used 'x' again here!
        print(x)  # Output: 10
        # print(x) # Original text had this line duplicated, output is still 10
        ```
        في هذا المثال، تم تغيير قيمة `x` داخل عبارة `if`[cite: 1171, 1172].
      * **الحل**: اختر أسماء مميزة للمتغيرات لتجنب هذه المشكلة[cite: 1173].

5.  **تجاهل الاستثناءات (Ignoring Exceptions)**

      * **المشكلة**: الاستثناءات هي أحداث تعطل التدفق الطبيعي للكود. الفشل في التعامل معها يمكن أن يؤدي إلى تعطل البرنامج[cite: 1175, 1176]. مثال شائع هو `ZeroDivisionError`[cite: 1177].
        ```python
        # x = 10
        # y = 0
        # result = x / y  # ZeroDivisionError
        ```
      * **الحل**: استخدم كتل `try-except` للتعامل مع الاستثناءات بأمان[cite: 1180, 1181].
        ```python
        x = 10
        y = 0
        try:
            result = x / y
        except ZeroDivisionError:
            print("Error: Division by zero")
        ```

###### مخاطر متقدمة وأفضل الممارسات:

  * **الواردات الدائرية (Circular Imports)**: تحدث عندما يحتاج ملفان إلى استخدام الكود من بعضهما البعض، مما يؤدي إلى حلقة[cite: 1186, 1195]. ستتعلم كيفية تجنبها لاحقًا[cite: 1198].
  * **إدارة الذاكرة (Memory Management)**: الاستخدام غير الفعال للذاكرة يمكن أن يسبب مشاكل، خاصة في البرامج الكبيرة[cite: 1199, 1202].
    ```python
    # مثال على استهلاك الذاكرة (لا تقم بتشغيله لفترة طويلة)
    # my_string = ""
    # lots_of_as = "a" * 1000000000
    # while True:
    #     my_string = my_string + lots_of_as  # Add 1 billion "a" characters
    ```
  * **الاختبار وتصحيح الأخطاء (Testing and Debugging)**: اختبر الكود بدقة واستخدم أدوات تصحيح الأخطاء لتحديد الأخطاء وإصلاحها[cite: 1203, 1204]. تطوير مجموعة قوية من الاختبارات الآلية يساعد على ضمان الاتساق[cite: 1212].

###### مقدمة للقوائم

القوائم هي هياكل بيانات أساسية في Python، تعمل كحاويات متعددة الاستخدامات لتخزين مجموعات من العناصر[cite: 1215]. يمكنها الاحتفاظ بأنواع بيانات متنوعة، بما في ذلك الأرقام والسلاسل والقيم المنطقية وحتى القوائم الأخرى[cite: 1216].

###### إعلان القوائم:

  * **قائمة فارغة**: `new_list = []` [cite: 1219, 1220]
  * **قائمة أرقام**: `primes = [2, 3, 5, 7, 11, 13]` [cite: 1221]
  * **قائمة سلاسل**: `vowels = ["a", "e", "i", "o", "u"]` [cite: 1222]
  * **قائمة قيم منطقية**: `coin_flips = [True, False, True, True, False]`
  * **قائمة بقيم مختلطة**: `student_record = ["Aashvi", 24, "Computer Science", 3.8, True]` [cite: 1223, 1225]

##### بدء استخدام القوائم

مثال قائمة التسوق:

```python
grocery_list = ["milk", "hummus", "bread", "cheese", "apples"]
```

###### قوائم الطباعة: عرض المحتويات

يمكن طباعة القائمة مباشرة باستخدام `print()`[cite: 1228].

```python
print(grocery_list)
#output:-['milk', 'hummus', 'bread', 'cheese', 'apples']
```

###### الفهرسة (Indexing)

يتم تمثيل كل عنصر في القائمة برقم فهرس، يبدأ من 0[cite: 1231, 1232].
للوصول إلى "milk": `print(grocery_list[0])`[cite: 1233].
لمعرفة عدد العناصر (طول القائمة)، استخدم `len()`[cite: 1233].

```python
print(len(grocery_list))
```

للوصول إلى العنصر الأخير: `print(grocery_list[len(grocery_list) - 1])` [cite: 1236] أو:

```python
last_element = len(grocery_list) - 1
print(grocery_list[last_element])
```

###### التقسيم (Slicing): الوصول إلى جزء من قائمتك

يتيح لك استخراج "شريحة" من القائمة[cite: 1238].
للحصول على أول عنصرين: `print(grocery_list[0:2])` (يشمل الفهرس 0 حتى الفهرس 2 ولكن لا يتضمنه)[cite: 1238, 1239].

###### فهم القائمة (List Comprehension): نهج موجز

طريقة موجزة لإنشاء قوائم جديدة بناءً على القوائم الموجودة، غالبًا ما تستخدم للتكرار (يشار إليها أيضًا بحلقة "for each")[cite: 1240, 1241].
مثال لحساب متوسط درجات الاختبار:

```python
average_score = 0
exam_scores = [85, 90, 78, 92, 88]
total_score = 0
for score in exam_scores: # List comprehension
    total_score = total_score + score
average_score = total_score / len(exam_scores)
print("The average score is:", average_score)
```

الناتج سيكون 86.6[cite: 1242].

###### المزيد من القوائم المتقدمة

  * **الأخطاء الشائعة في القوائم**:
      * `IndexError`: محاولة الوصول إلى فهرس غير موجود[cite: 1245, 1246].
      * `ValueError`: محاولة استخدام طريقة قائمة بقيمة غير منطقية (مثل إزالة عنصر غير موجود)[cite: 1247, 1248].
  * **القوائم ثنائية الأبعاد (2D Lists)**: قوائم داخل قوائم، لتمثيل هياكل مثل الشبكات[cite: 1249, 1250, 1251].
    ```python
    grid = [
        [1, 2, 3],
        [4, 5, 6],
        [7, 8, 9]
    ]
    print(grid[1][2]) # للوصول إلى الرقم 6
    ```
  * **عمليات القائمة المتقدمة (طرق القائمة)**:
      * `len(list)`: تُرجع عدد العناصر[cite: 1255].
      * `list.append(x)`: يضيف `x` إلى نهاية القائمة[cite: 1256].
      * `list.insert(i, x)`: يُدرج `x` في الفهرس `i`[cite: 1257].
      * `list.remove(x)`: يزيل أول ظهور لـ `x`[cite: 1259].
      * `list.sort()`: يفرز العناصر بترتيب تصاعدي (للأرقام أو السلاسل فقط، وليس مزيجًا)[cite: 1260, 1261].
      * `list.reverse()`: يعكس ترتيب العناصر[cite: 1262].
      * `list.count(x)`: يحسب عدد مرات ظهور `x`[cite: 1263].

القوائم في Python هي حاويات بيانات أساسية لتنظيم وإدارة مجموعات من العناصر[cite: 1276, 1277].

2.  **الطبيعة القابلة للتغيير (Mutability)**: القوائم ديناميكية ويمكن تعديلها بعد إنشائها (إضافة، إزالة، أو تغيير العناصر)[cite: 1283, 1286, 1287]. هذا يجعلها مثالية للبيانات المتطورة (مثل سجلات العملاء أو مستويات المخزون)[cite: 1288].
3.  **التنوع (Versatility)**: يمكن للقوائم تخزين مجموعة متنوعة من أنواع البيانات (أرقام، سلاسل، قيم منطقية، قوائم أخرى)، مما يسمح بإنشاء هياكل بيانات معقدة[cite: 1283, 1289, 1290].

  * **وسائل التواصل الاجتماعي**: لتخزين ملفات تعريف المستخدمين، والمشاركات، والتعليقات، والإعجابات، وتحليل المعلومات لخلاصات الأخبار والتوصيات[cite: 1294].

###### عمليات القائمة الأساسية:

3.  **التقطيع (Slicing)**: استخراج جزء من القائمة عن طريق تحديد فهرس البداية والنهاية[cite: 1298, 1305]. يتيح العمل مع مجموعات فرعية محددة من البيانات[cite: 1306]. مثال: تقسيم قائمة بريدية حسب الأحياء بناءً على نطاق من الشوارع[cite: 1307, 1308].

## الوحدة 3: بناء الكود المنظم والقابل لإعادة الاستخدام في بايثون: الدوال، الأصناف، والوحدات

### موضوع 1:-

##### مقدمة إلى الدوال (Functions)

في تطوير البرمجيات، تعتبر الكفاءة والتنظيم وإعادة الاستخدام أمورًا بالغة الأهمية. تقدم Python أداة قوية لتحقيق ذلك وهي الدوال (Functions).

###### ما هي الدالة (Function)؟

الدوال هي كتل من التعليمات البرمجية قابلة لإعادة الاستخدام تؤدي مهام محددة. إنها تحتوي على مجموعة من التعليمات، مما يسمح لك بتنفيذ هذه التعليمات عدة مرات حسب الحاجة في برنامجك، دون الحاجة إلى إعادة كتابة التعليمات البرمجية في كل مرة. يمكن اعتبارها برامج مصغرة داخل البرنامج الأكبر.

###### لماذا نستخدم الدوال؟

تعتبر الدوال مهمة لأنها توفر النظام والكفاءة وإعادة الاستخدام للكود، مما يؤثر على برمجة Python في أربعة مجالات رئيسية:

- **تنظيم الكود (Code Organization)**: تقوم الدوال بتقسيم المهام المعقدة إلى أجزاء أصغر وأكثر قابلية للإدارة، مما يجعل الكود أسهل في القراءة والفهم والصيانة.
- **إعادة الاستخدام (Reusability)**: بمجرد تحديد دالة معينة، يمكنك استدعاؤها عدة مرات في جميع أنحاء البرنامج، مما يقلل من تكرار الكود ومخاطر الأخطاء.
- **التجريد (Abstraction)**: تعمل الدوال كطبقة من التجريد، حيث تخفي التفاصيل المعقدة للمهمة خلف واجهة بسيطة، مما يتيح لك التركيز على المنطق عالي المستوى.
- **التعاون (Collaboration)**: تعزز الدوال التعاون من خلال إنشاء وحدات معيارية قائمة بذاتها من التعليمات البرمجية يمكن لأعضاء الفريق المختلفين العمل عليها بشكل مستقل.

###### أنواع الدوال في Python:

1. **دوال مدمجة (Built-in Functions)**: تأتي Python مع عدد من الدوال المحددة مسبقًا، مثل `print()`, `len()`, `input()`.
2. **دوال معرّفة من قبل المستخدم (User-defined Functions)**: هي الدوال التي تقوم بإنشائها لتنفيذ مهام محددة في برنامجك.
3. **دوال لامدا (Lambda Functions)**: هي دوال مجهولة ذات سطر واحد تُستخدم غالبًا للعمليات البسيطة أو كوسيطات لوظائف أخرى.

###### الصيغة الأساسية للدالة:

Python

```
def function_name(parameters):
    """Docstring explaining the function's purpose."""
    # Code to perform the task
    return # Optional return value
```

- `def`: الكلمة الرئيسية لتعريف الدالة.
- `function_name`: الاسم الوصفي الذي تعطيه للدالة.
- `parameters`: قيم الإدخال التي يمكن للدالة قبولها (بين قوسين).
- `Docstring`: سلسلة نصية تشرح الغرض من الدالة (داخل علامات اقتباس ثلاثية).
- `return`: كلمة أساسية اختيارية تُستخدم لإرسال قيمة مرة أخرى إلى جزء البرنامج الذي استدعى الدالة.

### موضوع 2:-

##### مقدمة إلى الأصناف (Classes)

إذا كان عليك إنشاء أسطول من روبوتات التوصيل لشركة لوجستية، فهل تفضل بناء كل روبوت من الصفر أو استخدام خطة رئيسية (master plan) تتيح لك إنشاء العديد من الروبوتات بفعالية بميزات مخصصة؟ في Python، تُعرف هذه الخطة الرئيسية بالفئة (Class).

###### ما هي الأصناف (Classes)؟

الأصناف هي كيانات رقمية تلخص مجموعة من السمات (attributes) أو البيانات والأساليب (methods) أو الوظائف. يحددون معًا خصائص وسلوكيات الأشياء (objects). إنها تشبه القالب أو المخطط الذي يمكنك استخدامه لإنشاء كائنات متعددة بناءً على بيانات ووظائف الفئة.

###### قدرات الأصناف:

توفر الأصناف ثلاث قدرات مهمة لتنظيم التعليمات البرمجية وإعادة استخدامها وتوسيعها:

1. **التغليف (Encapsulation)**: تجمع الأصناف البيانات والوظائف التي تعمل على تلك البيانات معًا، مما يؤدي إلى إنشاء وحدات قائمة بذاتها من التعليمات البرمجية.
2. **الوراثة (Inheritance)**: يمكن للأصناف أن ترث السمات والأساليب من الأصناف الأخرى، مما يشكل علاقة هرمية. يتيح لك هذا إنشاء أصناف متخصصة تعتمد على أساس أصناف أكثر عمومية.
3. **تعدد الأشكال (Polymorphism)**: يسمح للكائنات من فئات مختلفة بالاستجابة لنفس استدعاء الوظيفة بطريقتها الفريدة، مما يوفر المرونة والقدرة على التكيف في التعليمات البرمجية الخاصة بك.

###### مكونات الفئة في Python:

- **السمات (Attributes)**: هي متغيرات تخزن البيانات الخاصة بكل كائن من الفئة. تحدد خصائص الكائن.
- **الأساليب (Methods)**: هي الوظائف في الفئة التي تمثل الإجراءات أو السلوكيات التي يمكن أن يقوم بها الكائن.
- **المُنشئ (Constructor)**: هو أسلوب خاص (`__init__`) يتم استدعاؤه تلقائيًا عند إنشاء كائن جديد للفئة. يقوم بتهيئة سمات الكائن وتعيين القيم الأولية.

###### إنشاء الكائنات (Object Instantiation):

بمجرد تحديد فئة، يمكنك إنشاء مثيل (instance) لتلك الفئة، وهي عملية تسمى إنشاء مثيل (instantiation).

أمثلة واقعية للأصناف:

- **إدارة المخزون**: قد تمثل فئة `Product` كل منتج فردي في النظام، بسمات مثل الاسم والسعر ومستوى المخزون، وأساليب مثل `UpdateStock` أو `GenerateReport`.
- **تطوير الألعاب**: في الألعاب، تجلب الأصناف الشخصيات والأعداء وعناصر اللعبة الأخرى إلى الحياة. قد تحتوي فئة `Character` على سمات مثل الصحة والقوة والموضع، وأساليب مثل `Attack`, `Move`, `TakeDamage`.

### موضوع 3:-

##### مبدأ DRY والوظائف: أساس الكود الفعال

في تطوير البرمجيات، ظهر مبدأ **DRY (Don't Repeat Yourself)** كفلسفة أساسية، يدعو إلى التخلص من التعليمات البرمجية الزائدة عن الحاجة ودمج التعليمات المتكررة في كتلة واحدة قابلة لإعادة الاستخدام. الوظائف (Functions) هي الوسيلة المثالية لتنفيذ هذا المبدأ.

###### مبدأ DRY (Don't Repeat Yourself):

يهدف هذا المبدأ إلى تقليل تكرار المعلومات من جميع الأنواع، وهو مهم بشكل خاص في نظام متعدد المستويات. يتعلق الأمر بتقليل مساحة السطح للأخطاء والتناقضات ومشاكل الصيانة.

###### الوظائف (Functions):

الوظائف هي وحدات قائمة بذاتها من التعليمات البرمجية مصممة لأداء مهام محددة. تعمل كأفعال في مفردات الكود، وتتضمن إجراءات مثل `calculate_area` أو `validate_address`.

- **الحجج (Arguments/Inputs)**: هي القيم التي تقدمها للدالة عند استدعائها لمعالجتها.
- **قيم الإرجاع (Return Values/Outputs)**: هي النتيجة التي تنتجها الدالة بعد معالجة المدخلات.

مثال لدالة `calculate_area`:



```Python
def calculate_area(length, width):
  area = length * width
  return area

# Example usage:
length = 5
width = 3
rectangle_area = calculate_area(length, width) # Function call, providing inputs
print(f"The area of the rectangle is: {rectangle_area}") # Using the output
```

###### فوائد إعادة استخدام الكود من خلال الوظائف:

- **قابلية التوسع (Scalability)**: تسمح بتقسيم المشكلات الكبيرة إلى أجزاء أصغر وأكثر قابلية للإدارة.
- **قابلية الصيانة (Maintainability)**: تسهل عزل التغييرات وتصحيح الأخطاء في مكونات محددة.
- **العمل الجماعي (Teamwork)**: تمكن أعضاء الفريق المختلفين من العمل على وظائف مختلفة بشكل مستقل.
- **الاختبار (Testing)**: الوظائف هي وحدات قابلة للاختبار بطبيعتها.
- **قابلية القراءة (Readability)**: تحسن قابلية قراءة الكود من خلال توفير بنية واضحة.
- **تصحيح الأخطاء (Debugging)**: تساعد في تصحيح الأخطاء عن طريق عزل المشكلات المحتملة.

###### موازنة DRY مع التطبيق العملي:

في حين أن DRY هو دليل قيم، إلا أن الإفراط في تطبيقه يمكن أن يؤدي إلى تعقيد غير ضروري. المفتاح هو تحقيق التوازن بين إعادة الاستخدام والبساطة.

###### الوظائف في نظام Python البيئي الحديث:

تعتبر الوظائف أساسية لمكتبة Python القياسية والنظام البيئي الواسع لحزم الطرف الثالث. تتضمن الميزات التي تعزز قوة الوظائف:

- **المزخرفات (Decorators)**
- **المولدات (Generators)**

###### فوائد الدوال المضمنة:

- **`input()`**: تطالب المستخدم بإدخال البيانات وتُرجع المدخلات كسلسلة.
    - مثال: يمكن استخدامها في تطبيق إدخال البيانات لجمع معلومات المستخدم.

###### لماذا الوحدات مهمة؟

### موضوع 6:-

##### إنشاء الدوال المخصصة (Custom Functions)

بالإضافة إلى الدوال المضمنة، تمنحك Python القدرة على كتابة دوالك المخصصة لتلبية احتياجات مشاريعك الخاصة.

###### مكونات الدالة المخصصة:

- **الحجج (Arguments)**: هي قيم الإدخال التي تمررها إلى دالة عند استدعائها. توفر البيانات اللازمة للدالة لأداء مهمتها.
    - مثال: دالة لحساب صافي راتب الموظف قد تأخذ الراتب الإجمالي ومعدل الضريبة والخصومات كحجج.
- **قيم الإرجاع (Return Values)**: هي المخرجات أو النتيجة التي تم إنشاؤها بواسطة الدالة بعد معالجة الحجج وتنفيذ مهمتها.
    - مثال: دالة تتحقق من صحة كلمة المرور قد تُرجع قيمة `True` أو `False`.

###### خطوات كتابة الدالة في Python:

1. **حدد الدالة (Define the Function)**: ابدأ بالكلمة الأساسية `def`، متبوعة باسم الدالة (باستخدام snake_case إذا كان أكثر من كلمة)، ثم الأقواس التي تحتوي على الحجج، وأخيراً نقطتين (`:`).
2. **أضف سلسلة مستندات (Add a Docstring)**: وصف واضح وموجز لما تقوم به الدالة، والغرض منها، والحجج، وقيمة الإرجاع (داخل علامات اقتباس ثلاثية).
3. **اكتب الكود (Write the Code)**: في الكتلة ذات المسافة البادئة التي تتبع تعريف الدالة، اكتب الكود الذي ينفذ المهمة المطلوبة.
4. **أرجع قيمة (Return a Value)**: (اختياري) استخدم عبارة `return` لإرسال قيمة مرة أخرى إلى رمز الاستدعاء.
5. **احفظ الدالة (Save the Function)**.

###### استدعاء الدوال (Calling Functions):

بعد تحديد الدالة، يمكنك استدعاؤها من أجزاء أخرى من البرنامج لتنفيذ التعليمات البرمجية المضمنة فيها. يؤدي استدعاء الدالة إلى إظهار قابليتها لإعادة الاستخدام، ويعزز الوحدات النمطية، ويمنحها المرونة.

### موضوع 7:-

##### فهم نطاق المتغير (Variable Scope)

يحدد نطاق المتغير (Variable Scope) مكان الوصول إلى المتغير ومدة وجوده داخل التعليمات البرمجية. هناك نوعان رئيسيان من النطاقات: محلي وعالمي.

###### النطاق المحلي (Local Scope):

المتغيرات التي يتم إنشاؤها داخل دالة (محلية) لا يمكن الوصول إليها إلا داخل تلك الدالة. بمجرد انتهاء الدالة، تختفي هذه المتغيرات. هذا يساعد في الحفاظ على ترتيب الأشياء ويمنع التغييرات العرضية.

###### النطاق العالمي (Global Scope):

المتغيرات العالمية هي تلك التي يمكن الوصول إليها عبر التعليمات البرمجية بأكملها؛ فهي لا تقتصر على أي وظيفة معينة. يجب توخي الحذر مع المتغيرات العالمية، لأن تغييرها في وظيفة واحدة يمكن أن يكون له عواقب غير مقصودة في أجزاء أخرى من الكود.

### موضوع 8:-

##### تفصيل نطاق المتغير: المحلي والعالمي وقاعدة LEGB

يشبه نطاق المتغير تخطيط مستودع منظم جيدًا، حيث يحدد أي أجزاء من الكود يمكنها الوصول إلى "صناديق" (متغيرات) معينة.

###### النطاق المحلي (Local Scope): المتغيرات ذات الوصول المحدود

توجد المتغيرات المحلية داخل مناطق عمل مخصصة (مثل الدوال أو كتل التعليمات البرمجية) وهي مخصصة حصريًا للمهام داخل مساحة العمل تلك.

- **التغليف (Encapsulation)**: تحتفظ بالبيانات والعمليات بدقة داخل مساحات عملها.
- **تجنب تضارب الأسماء (Avoiding Name Conflicts)**: يمكنك إعادة استخدام نفس أسماء المتغيرات في دوال مختلفة دون تعارض.
- **إدارة الذاكرة (Memory Management)**: يتم إنشاؤها عند بدء الدالة وتدميرها عند انتهائها، مما يضمن الاستخدام الفعال للذاكرة.

مثال للكود:

Python

```
def calculate_average(numbers):
    total = 0  # Local variable
    count = len(numbers)  # Local variable
    for num in numbers:
        total += num
    average = total / count
    return average

print(calculate_average([2, 5, 4, 1]))
```

###### النطاق العالمي (Global Scope): المتغيرات ذات الوصول الشامل

توجد المتغيرات العالمية في منطقة المستودع الرئيسية، ويمكن الوصول إليها وتعديلها بواسطة أي دالة داخل الوحدة. يتم تعريفها عادةً في بداية الوحدة (الملف).

مثال للكود:

Python

```
# Global variable
COMPANY_NAME = "Global Tech Solutions"

def print_welcome_message():
    print("Welcome to", COMPANY_NAME)

def print_contact_info():
    print("Contact us at info@", COMPANY_NAME.lower()) 

print_welcome_message()
# Output: Welcome to Global Tech Solutions
print_contact_info()
# Output: Contact us at info@globaltechsolutions
```

يجب استخدام المتغيرات العالمية بحكمة، خاصة للثوابت أو البيانات التي تحتاج إلى مشاركة عبر وظائف متعددة.

###### قاعدة LEGB: استراتيجية بحث Python للمتغيرات

عند استخدام متغير، تبحث Python عن قيمته وفقًا لقاعدة LEGB:

1. **L (Local)**: تتحقق أولاً داخل الدالة الحالية أو كتلة التعليمات البرمجية.
2. **E (Enclosing function locals)**: إذا لم يتم العثور عليه محليًا، فإنها تبحث في الدوال المتضمنة (للدوال المتداخلة).
3. **G (Global)**: بعد ذلك، تبحث في النطاق العالمي للوحدة.
4. **B (Built-in)**: أخيرًا، تتحقق من مساحة الاسم المضمنة في Python للدوال والكائنات المحددة مسبقًا.

###### النطاقات المتداخلة (Nested Scopes) والنطاق المرفق (Enclosing Scope)

في الدوال المتداخلة، يكون المتغير في الدالة الخارجية (النطاق المرفق) مرئيًا للدالة الداخلية، حتى لو لم يكن عالميًا.

مثال للكود:

```

    outer_var = "Hello from the outer function!"
    def inner_function():
        print(outer_var)  # Accessible from the enclosing scope
    inner_function()

outer_function()
```

###### أفضل الممارسات لنطاق المتغير:

- استخدم المتغيرات المحلية كلما أمكن ذلك.
- استخدم المتغيرات العالمية باعتدال.
- اختر أسماء ذات معنى للمتغيرات.
- لا تستخدم نفس الاسم للمتغيرات المحلية والعالمية لتجنب الارتباك.

##### أساسيات الأصناف (Classes) والبرمجة الشيئية (Object-Oriented Programming - OOP)

الأصناف هي حجر الزاوية في البرمجة الشيئية (OOP) في Python. إنها توفر مخططًا منظمًا لإنشاء الكائنات (objects)، وهي اللبنات الأساسية لتطبيقاتك.

###### فهم البرمجة الشيئية (OOP):

- **الكائنات (Objects)**: تمثل كيانات العالم الحقيقي أو المفاهيم المجردة داخل الكود. كل كائن هو وحدة قائمة بذاتها مع بياناتها الخاصة والقدرة على تنفيذ الإجراءات.
- **الأصناف (Classes)**: تعمل كمخططات أو قوالب لإنشاء الكائنات. تحدد الخصائص المشتركة (السمات) والسلوكيات (الأساليب).
- **التغليف (Encapsulation)**: يعني الاحتفاظ بتفاصيل كيفية عمل الكائن مخفية داخله. يقوم بتجميع البيانات والإجراءات التي تعمل على تلك البيانات داخل الفئة.
- **التجريد (Abstraction)**: يبسط الأنظمة المعقدة من خلال التركيز على الميزات والتفاعلات الأساسية وإخفاء التفاصيل غير الضرورية.
- **الوراثة (Inheritance)**: تتيح لك إنشاء أنواع جديدة من الكائنات التي تستند إلى الكائنات الموجودة، مع وراثة ميزاتها وإضافة ميزات جديدة.
- **تعدد الأشكال (Polymorphism)**: يعني أنه يمكنك التعامل مع أنواع مختلفة من الكائنات بطريقة مماثلة، طالما أنها تشترك في واجهة مشتركة.

###### أهمية الأصناف:

- **تنظيم الكود (Code Organization)**: تنشئ بنية واضحة للكود عن طريق تجميع البيانات والإجراءات ذات الصلة.
- **قابلية إعادة الاستخدام (Reusability)**: بمجرد تحديد فئة، يمكنك إنشاء مثيلات (كائنات) متعددة منها.

###### تشريح الفئة:

- **تعريف الفئة (Class Definition)**: يبدأ بالكلمة الأساسية `class` متبوعة باسم الفئة (عادةً ما يكون بـ CamelCase).
    
    Python
    
    ```

        # ... (class contents)
    ```
    
- **السمات (Attributes/Data)**: هي المتغيرات التي تخزن حالة الكائن أو خصائصه.
    
    Python
    
    ```
    class Car:
        def __init__(self, make, model, year, color):

            self.model = model
            self.year = year
            self.color = color
            self.fuel_level = 100  # Initial fuel level
    ```
    
- **الأساليب (Methods/Behaviors)**: هي وظائف محددة داخل الفئة تعمل على بيانات الكائن وتمثل الإجراءات التي يمكن أن يتخذها الكائن.
    
    Python
    

- **طريقة `__init__` (المنشئ/Constructor)**: هي طريقة خاصة تستدعيها Python تلقائيًا عند إنشاء كائن جديد. دورها الأساسي هو تهيئة سمات الكائن.

###### ميزات متقدمة للأصناف:

##### هياكل بيانات بايثون: ورقة الغش (Python Data Structures: Cheat Sheet)

هياكل البيانات المضمنة في Python هي حاويات متخصصة للاحتفاظ بالمعلومات وتنظيمها.

###### القوائم (Lists): حاويات البيانات متعددة الاستخدامات الخاصة بك

    - **إنشاء قائمة:** `my_list = [1, 2.1, "hello", [3, 4]]`
    - **الوصول إلى العناصر (الفهرسة - Indexing):** تبدأ الفهرسة من 0. `my_list[0]` يعطي العنصر الأول.
    - **استخراج المجموعات الفرعية (التقطيع - Slicing):** `my_list[1:3]` يستخرج العنصرين الثاني والثالث.
    - **توسيع القوائم (extend):** `tasks.extend(new_tasks)` لإضافة عناصر متعددة من قائمة أخرى.
    - **تعديل القوائم (قابلية التغيير - Mutability):** يمكن إضافة عناصر (`append()`, `insert()`)، إزالة عناصر (`remove()`, `pop()`)، أو إعادة ترتيبها (`sort()`, `reverse()`).
- **تطبيق عملي: قائمة المهام (To-Do List)**
    
    Python
    
    ```
    tasks = ["Buy groceries", "Finish report", "Call Mom"]
    tasks.append("Schedule dentist appointment")
    tasks.remove("Finish report")
    # tasks.sort()
    ```
    
    القوائم مناسبة عندما تحتاج إلى تخزين مجموعة من العناصر التي قد ترغب في تعديلها لاحقًا.

###### المجموعات (Tuples): مجموعات البيانات غير القابلة للتغيير

- **كيفية العمل مع Tuples:**
    - **إنشاء مجموعة:** `my_tuple = (10, 20, "python")`

    - **الثبات في العمل:** لا يمكن تعديل محتويات المجموعة بعد الإنشاء (لا يمكن استخدام `my_tuple[1] = 30` أو `my_tuple.append()`).
- **تطبيق العالم الحقيقي: الإحداثيات الجغرافية**
    
    Python
    
    ```
    coordinates = (37.7749, -122.4194) # خط العرض وخط الطول لسان فرانسيسكو
    ```
    
    تُستخدم عندما تريد التأكد من بقاء البيانات ثابتة.

###### المجموعات (Sets): مجموعات من العناصر الفريدة

- مجموعات غير مرتبة (unordered collections)، كل عنصر يظهر مرة واحدة فقط (uniqueness).
- **العمل مع مجموعات في بايثون:**
    - **إنشاء مجموعة:** `my_set = {1, 2, 3, 3}` ستنتج `{1, 2, 3}`.
    - **الطبيعة غير المرتبة:** لا يمكن الوصول إلى العناصر بالفهرس.
    - **العمليات الرئيسية:** `add()`, `remove()`, `union()`, `intersection()`, `difference()`.
- **تطبيق العالم الحقيقي: الإعجابات على وسائل التواصل الاجتماعي**
    
    Python
    
    ```
    liked_by = {12345, 67890, 98765} # User IDs
    ```
    
    مفيدة عند التعامل مع عناصر فريدة أو لتنفيذ عمليات المجموعة.

###### القواميس (Dictionaries): أزواج القيم الرئيسية (Key-Value Pairs)

- لتخزين المعلومات وتنظيمها عن طريق ربط كل جزء من البيانات بمفتاح فريد.
- يمكن أن تكون المفاتيح أرقامًا، سلاسل، أو حتى tuples. القيم يمكن أن تكون أي نوع بيانات.
- **العمل مع القواميس في بايثون:**
    - **إنشاء قاموس:** `my_dict = {"name": "Alice", "age": 30, "city": "New York"}`

    - **العمليات الشائعة:** `get()`, `items()`, `keys()`, `values()`, `update()`.
- **تطبيق العالم الحقيقي: كتالوج المنتج**

    - مثال: كميات العناصر في قائمة التسوق يمكن تحديثها. <!-- end list -->

### موضوع 4:-

##### القواميس (Dictionaries) في Python: نظرة متعمقة

القواميس في Python تعمل مثل دفتر عناوين عالي التقنية، حيث كل إدخال له `key` فريد مرتبط بـ `value` محددة.

###### خصائص القواميس الرئيسية:

1. **عمليات بحث فعالة (Efficient Lookups):** إدخال المفتاح يتيح الوصول المباشر والسريع للقيمة المقابلة، حتى في مجموعات البيانات الكبيرة. مهم لتطبيقات مثل محركات البحث أو تطبيقات ترجمة اللغة.
2. **قابلية التغيير (Mutability):** القواميس مرنة وقابلة للتكيف. يمكن إضافة أو إزالة أو تعديل أزواج `key-value` بسهولة. مناسبة لبيانات تتطور مع الوقت مثل كتالوج المنتجات أو ملفات تعريف المستخدمين على وسائل التواصل الاجتماعي.
3. **تمثيل العلاقات في العالم الحقيقي (Representing Real-World Relationships):** جيدة في نمذجة العلاقات بين أجزاء مختلفة من المعلومات. مثال: نظام إدارة علاقات العملاء (CRM) حيث يرتبط معرف العميل (مفتاح) بقيم معلوماتية متعددة (تفاصيل الاتصال، سجل الشراء).

###### وظيفة زوج القيمة-المفتاح (`key-value` pair):

- **المفتاح (Key):** يعمل كمعرف فريد لجزء معين من المعلومات (مثل ملصق على درج).
- **القيمة (Value):** هي البيانات الفعلية المرتبطة بالمفتاح (مثل الملفات في الدرج).
    - مثال قاعدة بيانات الموظفين: مفتاح "الاسم" قيمته الاسم الأول والأخير، مفتاح "القسم" قيمته القسم الذي يعمل به.

###### الطبيعة غير المرتبة (Unordered Nature):

- في الإصدارات القديمة من Python (قبل 3.7)، لم تكن القواميس تحافظ على ترتيب الإدخال.
- في Python 3.7 والإصدارات الأحدث، **تحافظ القواميس على ترتيب الإدراج**. هذا يعني أن الترتيب الذي تضيف به أزواج المفاتيح والقيم هو نفس الترتيب الذي سيتم به تكرارها أو عرضها. (ملاحظة: النص المصدر قد يحتوي على معلومة قديمة حول هذا).
- القواميس تعطي الأولوية للعلاقة بين المفاتيح والقيم بدلاً من ترتيبها (تاريخيًا).

###### عمليات بحث سريعة:

- القواميس مُحسَّنة لعمليات البحث السريعة باستخدام المفاتيح.
- مثال: كتالوج منتجات في متجر إلكتروني، حيث يُستخدم معرّف المنتج الفريد (SKU) كمفتاح للوصول السريع إلى معلومات المنتج.

### موضوع 6:-

##### المجموعات (Sets) في Python: التعامل مع العناصر الفريدة

المجموعات (Sets) في Python هي هياكل بيانات تُستخدم لتخزين مجموعات غير مرتبة من العناصر المميزة (unique members).

###### الخاصية الفريدة للمجموعات:

- كل عضو في المجموعة فريد.
- مزايا:
    - **اختبار العضوية الفعال (Efficient membership testing):** التحقق بسرعة مما إذا كان عضو معين موجودًا في المجموعة (باستخدام هياكل مثل جداول التجزئة - hash tables).
        - مثال: التحقق مما إذا كان فرد ما مشتركًا في قائمة بريدية.
        - مثال: تحليل إجابات استطلاع لتحديد الاستجابات الفريدة.
    - **القضاء على الأعضاء المكررين (Elimination of duplicate members):** إزالة القيم المكررة تلقائيًا.
        - مثال: تجميع قائمة الحاضرين لمؤتمر، مع التأكد من حساب كل شخص مرة واحدة.
        - مثال: تنظيف البيانات بإزالة المعلومات المتكررة من مصادر متعددة.

###### العمليات الرياضية على المجموعات:

- **الاتحاد (Union):** يجمع كل العناصر الفريدة من مجموعتين أو أكثر. `set1.union(set2)` أو `set1 | set2`.
    - مثال: دمج قائمتي عملاء من مصادر مختلفة، مع التأكد من ظهور كل عميل مرة واحدة.
- **التقاطع (Intersection):** يحدد العناصر المشتركة بين مجموعتين أو أكثر. `set1.intersection(set2)` أو `set1 & set2`.
    - مثال: مقارنة قائمتي مهارات مطلوبة لوظيفتين لتحديد المهارات الأساسية المشتركة.
    - مثال: العثور على العملاء الذين اشتروا منتجات من فئات مختلفة.

###### أمثلة واقعية لاستخدام المجموعات:

- **نظام CRM:** تخزين عناوين البريد الإلكتروني للعملاء، مع ضمان أن كل عنوان فريد.
- **تتبع المخزون:** التأكد من تمثيل كل منتج مرة واحدة فقط في سجلات المخزون.
- **وسائل التواصل الاجتماعي:** تحليل الاتجاهات عن طريق تصفية علامات التصنيف أو الكلمات الرئيسية المكررة.

### موضوع 8:-

##### هياكل البيانات: نظام تنظيم Python الخاص بك

هياكل البيانات هي حاويات متخصصة لتنظيم المعلومات.

###### القوائم (Lists): قائمة المهام المطلوبة

- مجموعة مرتبة من العناصر، كل عنصر له فهرس (position).
- مرنة ويمكن أن تخزن أنواع بيانات متنوعة.
- **الخصائص الرئيسية:**
    - **مرتبة (Ordered):** يتم الحفاظ على ترتيب إضافة العناصر.
    - **قابلة للتغيير (Mutable):** يمكن تعديل محتوياتها بعد الإنشاء.
    - **قابلة للفهرسة (Indexable):** يمكن الوصول إلى العناصر باستخدام فهرسها (يبدأ من 0).
- **عمليات شائعة:**
    - `append(item)`: يضيف عنصرًا إلى نهاية القائمة.
    - `insert(index, item)`: يُدرج عنصرًا في موضع محدد.
    - `pop()`: يزيل العنصر الأخير ويعيده.
    - `remove(item)`: يزيل أول ظهور للعنصر.
    - `del` statement: يحذف كائنات أو متغيرات أو عناصر محددة.

###### القواميس (Dictionaries): صناديق التخزين المصنفة

- تنظم البيانات باستخدام أزواج `key-value`.
- **الخصائص الرئيسية:**
    - **مرتبة (Ordered):** في Python 3.7+، تحافظ على ترتيب الإدراج. (ملاحظة: النص المصدر قد يشير إلى أنها غير مرتبة للإصدارات الأقدم).
    - **قابلة للتغيير (Mutable):** يمكن إضافة، تعديل، أو إزالة أزواج `key-value`.
    - **الوصول إلى القيمة-المفتاح (Key-Value Access):** بحث فعال باستخدام المفتاح للوصول إلى القيمة.
- **مثال الكود:**
    
    Python
    
    ```
    # Create a dictionary to store contact information
    contacts = {"Alice": "555-1234", "Bob": "555-5678", "Carol": "555-9012"}
    
    # Look up Bob's phone number
    bobs_phone = contacts["Bob"]
    print(bobs_phone)
    # Output: 555-5678
    
    # Add a new contact
    contacts["David"] = "555-4321"
    
    # Update Carol's phone number
    contacts["Carol"] = "555-2468"
    
    # Remove Alice's contact information
    del contacts["Alice"]
    
    # Print updated contacts
    print(contacts)
    ```
    
- **تطبيقات العالم الحقيقي:** دفاتر العناوين، كتالوجات المنتجات، إعدادات المستخدم، قواميس الترجمة.

###### المجموعات (Sets): مجموعة الطوابع الفريدة الخاصة بك

- تضمن تمييز جميع العناصر داخلها (uniqueness).
- **الخصائص الرئيسية:**
    - **غير مرتبة (Unordered):** لا يتم الحفاظ على ترتيب إضافة العناصر.
    - **قابلة للتغيير (Mutable):** يمكن إضافة وإزالة العناصر.
    - **فريدة (Unique):** كل عنصر يظهر مرة واحدة فقط.
- **عمليات شائعة:**
    - `add(item)`: يضيف عنصرًا إذا لم يكن موجودًا.
    - `remove(item)`: يزيل عنصرًا إذا كان موجودًا.
    - `union(other_set)`: يجمع مجموعتين.
    - `intersection(other_set)`: يجد العناصر المشتركة.
    - `difference(other_set)`: يجد العناصر الموجودة في المجموعة الأولى وليست في الثانية.
    - `issubset(other_set)`: يتحقق مما إذا كانت المجموعة الأولى مجموعة جزئية من الثانية.
    - `issuperset(other_set)`: يتحقق مما إذا كانت المجموعة الأولى مجموعة شاملة للثانية.
- **مثال الكود:**
    
    Python
    
    ```
    # Create a set of favorite programming languages
    languages={"Python","JavaScript","Java"}
    # Add "C++" to the set
    languages.add("C++")
    # Try to add "Python" again (it won't be added because it's a duplicate)
    languages.add("Python")
    print(languages)# Output: {'Python', 'C++', 'JavaScript', 'Java'} (order may vary)
    # Remove "Java"
    languages.remove("Java")
    # Create another set of languages
    web_languages={"JavaScript","HTML","CSS"}
    # Find common languages between the two sets
    common_languages=languages.intersection(web_languages)
    print(common_languages)# Output: {'JavaScript'}
    ```
    
- **تطبيقات العالم الحقيقي:** إزالة الإدخالات المكررة، التحقق من وجود عنصر، العثور على اهتمامات مشتركة، تحديد الكلمات الفريدة.

###### اختيار الأداة المناسبة:

- **القائمة:** لمجموعة مرتبة مع السماح بالتكرارات (إذا كان الترتيب مهمًا والتكرار مقبول).
- **القاموس:** لربط التصنيفات (المفاتيح) بالقيم والبحث السريع عنها.
- **المجموعة:** لضمان التفرد وتنفيذ عمليات المجموعة.

###### تحدي الكود: إنشاء قائمة

اكتب كود Python لأخذ قائمة أرقام كمدخل وإرجاع قائمة جديدة تحتوي فقط على الأرقام الفريدة من القائمة الأصلية، مع الحفاظ على ترتيبها الأصلي، ثم طباعة النتيجة.

- الكود الأصلي في المصدر يحتوي على خطأ في تحقيق الحفاظ على الترتيب الأصلي إذا تم استخدام `set()` ثم `list()` مباشرة. الطريقة الصحيحة للحفاظ على الترتيب تتطلب منطقًا إضافيًا.
    
    Python
    
    ```
    # Values provided (do not change)
    array=[1,2,2,3,1,4,5,3]
    # Corrected approach to get unique values while preserving order
    unique_array = []
    seen_elements = set()
    for item in array:
        if item not in seen_elements:
            unique_array.append(item)
            seen_elements.add(item)
    
    print(unique_array)
    ```

##### هياكل البيانات المرتبة مقابل غير المرتبة (Ordered vs. Unordered Data Structures)

- **هيكل البيانات المرتب (Ordered Data Structure):** يحافظ على العناصر بالترتيب الذي تمت إضافته أو إدراجه فيه. التسلسل مهم.
    - أمثلة: القوائم (Lists)، المجموعات (Tuples).
    - تُستخدم عندما يكون موضع كل عنصر مهمًا (قائمة مهام، خطوات عملية).
    - القوائم: تسمح بالوصول عبر الفهرس وعمليات تعتمد على التسلسل (الفرز، العكس).
    - المجموعات (Tuples): تشبه القوائم ولكنها غير قابلة للتغيير، تُستخدم لمجموعات بيانات ثابتة.
- **هيكل البيانات غير المرتب (Unordered Data Structure):** ترتيب العناصر غير مضمون. تعطي الأولوية لاسترجاع البيانات والعلاقة بينها بدلاً من موقعها.
    - أمثلة: المجموعات (Sets)، القواميس (Dictionaries).
    - المجموعات (Sets): مصممة للتحقق السريع من العضوية وتنفيذ عمليات المجموعة.
    - القواميس (Dictionaries): تخزن أزواج `key-value`، وتُستخدم لاسترداد المعلومات بسرعة بناءً
    - على معرف فريد (المفتاح). (ملاحظة: Python 3.7+ القواميس تحافظ على ترتيب الإدراج، لكن المفهوم العام لهياكل البيانات غير المرتبة لا يزال ينطبق على المجموعات و تاريخياً على القواميس).

###### لماذا يعتبر الترتيب مهمًا؟

- **المعالجة المتسلسلة (Sequential Processing):** عندما تحتاج إلى معالجة البيانات خطوة بخطوة أو الحفاظ على جدول زمني (مثل طلبات العملاء).
- **العلاقات المعتمدة على الترتيب (Order-Dependent Relationships):** عندما يمثل ترتيب العناصر علاقات ذات معنى (مثل قائمة تشغيل الموسيقى).
- **الوصول المباشر عبر الفهرس (Direct Access by Index):** في الهياكل المرتبة، لاسترداد عنصر معين أو تعديله في موقع معين.

###### تطبيقات واقعية للترتيب:

- **إدارة المشاريع:** تنظيم المهام حسب الأولوية أو تواريخ الاستحقاق (باستخدام القوائم).
- **أنظمة دعم العملاء:** معالجة الطلبات في قوائم الانتظار (نوع من هيكل البيانات المرتب).
- **تحليل بيانات السلاسل الزمنية:** الحفاظ على الترتيب الزمني لتحديد الاتجاهات.

###### تأثير الترتيب على الوصول إلى البيانات واسترجاعها:

- **الهياكل المرتبة:** وصول مباشر عبر الفهرس، سهولة التكرار بالترتيب الأصلي.
- **الهياكل غير المرتبة:**
    - المجموعات (Sets): تحقق سريع من العضوية.
    - القواميس (Dictionaries): استرداد سريع للقيم باستخدام المفاتيح.

##### الفرز (Sorting) في قوائم Python

الفرز هو عملية إعادة ترتيب العناصر داخل قائمة بناءً على قيمها.

###### أهمية الفرز:

- **تنظيم البيانات:** مثل ترتيب الكتب على رف.
- **تحليل البيانات:** يساعد في تحديد الاتجاهات، القيم المتطرفة، والأنماط.
- **إدارة المهام:** يساعد في تحديد أولويات المهام.

###### خوارزميات الفرز (Sorting Algorithms):

الخوارزميات هي الإجراءات التي تقارن وتعيد ترتيب العناصر.

1. **فرز الفقاعات (Bubble Sort):**
    - يتصفح القائمة مرارًا، يقارن الأزواج المتجاورة ويبدلها إذا كانت بالترتيب الخاطئ.
    - ينقل ببطء أكبر/أصغر العناصر إلى النهاية.
    - مثال: فريق ينظم مهام المشروع حسب الأولوية بمقارنة المهام القريبة.
2. **فرز التحديد (Selection Sort):**
    - يبحث عن أصغر/أكبر عنصر في الجزء غير المفرز من القائمة ويستبدله بالعنصر الأول في ذلك الجزء.
    - يتكرر للأجزاء غير المفرزة حتى يتم فرز كل شيء.
    - مثال: تنظيم رف كتب بالعثور على الكتاب الذي ينتمي للموضع التالي ووضعه.
3. **الفرز السريع (Quicksort):**
    - يقسم القائمة إلى مجموعتين بناءً على عنصر محوري (pivot).
    - يفرز كل مجموعة بشكل منفصل ويكرر العملية.
4. **الفرز المخصص (Custom Sorting) في Python:**
    - يمكن تحديد معايير الفرز (باستخدام `key` في دالة `sort()` أو `sorted()`).
    - فرز الكائنات بناءً على سمات محددة.
    - فرز القواميس أو قوائم Tuples بناءً على مفاتيح معينة.

###### فوائد الفرز:

- **الوضوح والنظام:** يحول البيانات الفوضوية إلى بيانات منظمة.
- **بحث فعال:** يتيح استخدام خوارزميات بحث فعالة مثل البحث الثنائي (binary search) على القوائم المفرزة.
- **تحليل بيانات أفضل:** يكشف عن الأنماط والاتجاهات والقيم المتطرفة.
- **تجربة مستخدم محسنة:** في تطبيقات مثل المتاجر الإلكترونية (فرز المنتجات).

##### البحث في القوائم (Searching Lists) في Python

البحث هو عملية تحديد موقع عنصر معين داخل قائمة.

###### مبادئ أساسية:

- **القوائم (Lists):** مجموعات مرتبة من العناصر يمكن الوصول إليها عبر الفهرس.
- **البحث (Searching):** تحديد موقع عنصر معين.
- **الخوارزميات (Algorithms):** إجراءات خطوة بخطوة لحل المشكلات (مثل البحث).
- **الكفاءة (Efficiency):** سرعة إكمال الخوارزمية لمهمتها (مهم للقوائم الكبيرة).

###### خوارزميات البحث:

1. **البحث الخطي (Linear Search):**
    - خوارزمية مباشرة تتحقق من كل عنصر في القائمة بالتتابع.
    - الكود: يتكرر عبر كل عنصر ويقارنه بالهدف؛ إذا وجد تطابقًا، يُرجع الفهرس، وإلا يُرجع -1 (أو يثير استثناء).
    - **الميزات:** بسيط، يعمل مع أي قائمة (حتى لو لم تكن مفرزة).
    - **العيوب:** يمكن أن يكون بطيئًا للقوائم الكبيرة جدًا.
    - **متى يُستخدم:** قائمة صغيرة، قائمة غير مفرزة، أو قد تحتاج للبحث في بداية القائمة فقط.
    - يمكن تحسينه بتقسيم القائمة إلى شرائح ثم البحث.
2. **البحث الثنائي (Binary Search):**
    - يتطلب أن تكون القائمة مفرزة.
    - يقسم القائمة بشكل متكرر إلى النصف ويتحقق من العنصر الأوسط.
    - الكود: يحتفظ بمؤشرين `low` و `high` لبداية ونهاية جزء البحث. يحدد العنصر الأوسط `mid` ويقارنه بالهدف. بناءً على المقارنة، إما يجد الهدف أو يضيق نطاق البحث بتعديل `low` أو `high`.
    - **الميزات:** سريع للغاية للقوائم الكبيرة والمفرزة.
    - **متى يُستخدم:** قائمة كبيرة ومفرزة، عندما تجري العديد من عمليات البحث في نفس القائمة، والسرعة أمر بالغ الأهمية.
    - تطبيقات: أبحاث الجينوم، تحليل البيانات المالية، محركات البحث، موازنة تحميل الخادم.

###### تحديد الخوارزمية المناسبة:

- **قائمة صغيرة أو غير مفرزة:** بحث خطي.
- **قائمة كبيرة ومفرزة:** بحث ثنائي.

###### أدوات وتقنيات إضافية للبحث في Python:

- **`in` operator:** طريقة بسيطة للتحقق من وجود عنصر.
- **فهم القائمة (List Comprehensions):** لإنشاء قوائم جديدة بناءً على معايير البحث.
- **طريقة `index()`:** تُرجع فهرس أول ظهور لعنصر معين (تثير `ValueError` إذا لم يُعثر عليه).
- **وحدة `bisect`:** توفر آليات فعالة لإدارة القوائم المفرزة، بما في ذلك الإدراج السريع مع الحفاظ على الترتيب.

    - خوارزمية مختلفة مثل **Quicksort**: أسرع بكثير، خاصة للمجموعات الكبيرة.
        
        - تختار عنصر "محوري" (pivot) وتقسم الباقي إلى أقل من/يساوي المحور وأكبر من المحور.
        - تفرز القوائم الفرعية بشكل متكرر ثم تدمجها. <!-- end list -->
        
        Python
        
        ```
        def quicksort(cards):
            if len(cards) < 2:
                return cards  # Base case: Already sorted if 0 or 1 element
            else:
                pivot = cards[0]  # Choose first card as pivot
                less = [i for i in cards[1:] if i <= pivot]
                greater = [i for i in cards[1:] if i > pivot]
                return quicksort(less) + [pivot] + quicksort(greater)
        ```
        
- **تدوين Big O Notation:** طريقة لقياس كيفية تغير أداء الخوارزمية مع زيادة كمية البيانات. يساعد في التنبؤ بأداء الخوارزمية.

###### تأثير الخوارزميات:

- **تطبيقات الملاحة (GPS):** تستخدم خوارزميات الرسم البياني (graph algorithms) لإيجاد أسرع طريق.
- **توصيات البث (Netflix, Spotify):** تستخدم خوارزميات التعلم الآلي (machine learning algorithms) لتحليل سجل المشاهدة/الاستماع وتقديم توصيات.
- **التصوير الطبي:** خوارزميات معالجة الصور (image processing algorithms) تحلل الأشعة لاكتشاف الأمراض.

### موضوع 17:-

##### اختيار هيكل البيانات المناسب

اختيار هيكل البيانات الصحيح يؤثر على كفاءة وسرعة ووضوح الكود.

###### هياكل البيانات الأساسية في Python:

- **القوائم (Lists):** مجموعات متنوعة ومرتبة. مثالية للتسلسلات والبيانات الديناميكية.
- **القواميس (Dictionaries):** أزواج `key-value` لعمليات بحث سريعة. جيدة للارتباطات بين البيانات.
- **المجموعات (Sets):** مجموعات غير مرتبة من العناصر الفريدة. مثالية لإزالة التكرارات أو التحقق من العضوية.
- **المجموعات (Tuples):** مجموعات مرتبة وغير قابلة للتغيير. تُستخدم غالبًا لتمثيل مجموعات ثابتة من البيانات.

###### عوامل رئيسية في اختيار بنية البيانات:

1. **نوع البيانات التي تعمل معها؟**
    - **مرتبة (Ordered)** (تسلسل محدد مثل خطوات وصفة) -> قوائم، tuples.
    - **غير مرتبة (Unordered)** (الترتيب لا يهم مثل مجموعة معرفات مستخدمين فريدة) -> مجموعات.
    - **قابلة للتغيير (Mutable)** (يمكن تغييرها بعد الإنشاء مثل عناصر عربة تسوق) -> قوائم، قواميس.
    - **غير قابلة للتغيير (Immutable)** (لا يمكن تغييرها بعد الإنشاء مثل أيام الأسبوع) -> tuples.
2. **ما العمليات التي ستقوم بها بشكل متكرر؟**
    - **إضافة/إزالة:** قوائم (فعالة في النهاية)، مجموعات وقواميس (إضافة/إزالة سريعة للعناصر الفريدة/حسب المفتاح).
    - **بحث:** قواميس (بحث سريع حسب المفتاح)، مجموعات (فعالة للتحقق من العضوية).
    - **فرز:** قوائم (تدعم خوارزميات فرز فعالة).
3. **ما مقدار البيانات التي تتعامل معها؟**
    - **مجموعات بيانات صغيرة:** اختلافات الأداء قد لا تكون ملحوظة. اختر الأسهل للعمل.
    - **مجموعات بيانات كبيرة:** الأداء حاسم. البحث في قائمة يمكن أن يكون أبطأ بكثير من البحث في قاموس.
4. **ما هي متطلبات تعقيد الوقت والمكان (Time and Space Complexity) لتطبيقك؟**
    - **تعقيد الوقت:** كيف يتغير وقت التنفيذ مع حجم البيانات (O(log n), O(n)).
    - **تعقيد المكان:** مقدار الذاكرة المستهلكة.

###### نصائح عملية وسيناريوهات:

- **قائمة ديناميكية للعناصر (Dynamic list of items):** (مثل قائمة تشغيل موسيقى، يمكن إضافة وإزالة الأغاني) -> **قائمة (List)**.
- **عمليات بحث سريعة بواسطة معرف فريد (Quick lookups by unique identifier):** (مثل معلومات ملف تعريف مستخدم باستخدام اسم المستخدم) -> **قاموس (Dictionary)**.
- **إزالة التكرارات (Removing duplicates):** (مثل تنظيف قائمة عناوين بريد إلكتروني) -> **مجموعة (Set)**.
- **حماية البيانات الثابتة (Protecting fixed data):** (مثل إحداثيات جغرافية لا يجب أن تتغير) -> **Tuple**.

###### مثال الكود: مقارنة البحث عن القائمة والقاموس

(لم يتم توفير الكود في النص لهذه النقطة مباشرة، ولكن الفكرة هي أن البحث في القاموس أسرع بشكل عام).

###### هياكل بديلة (More Specialized Structures):

- **`deque` (من `collections`):** للإضافة والحذف الفعال من كلا الطرفين (قائمة انتظار مزدوجة).
    - متى تُستخدم: تنفيذ قوائم الانتظار أو الأكوام، محاكاة الخطوط، وظيفة التراجع/الإعادة.
- **`heapq` (من `heapq`):** لقوائم الانتظار ذات الأولوية (priority queues)، حيث يكون العنصر الأصغر/ذو الأولوية القصوى دائمًا في المقدمة.
    - متى تُستخدم: جدولة المهام، خوارزمية Dijkstra.
- **`Counter` (من `collections`):** لحساب تكرارات العناصر في تكرار (iterable).
    
    - متى تُستخدم: تحليل تردد الكلمات، حساب حدوث العناصر، العثور على العناصر الأكثر شيوعًا. <!-- end list -->
    
    Python
    
    ```
    from collections import deque, Counter
    
    # Deque example
    queue = deque()
    queue.append("task1")
    queue.append("task2")
    print(queue.popleft())  # Output: task1
    
    # Counter example
    text = "This is a sample text with some repeated words words"
    word_counts = Counter(text.split())
    print(word_counts)  # Output: Counter({'words': 2, 'This': 1, 'is': 1, ...})
    ```
